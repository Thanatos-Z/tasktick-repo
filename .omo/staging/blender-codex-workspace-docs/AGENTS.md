# Codex + Blender 工作区规则

## 适用范围与沟通

- 本文件适用于当前 Blender 工作区及其按需创建的子目录；默认使用简体中文沟通和交付。
- 修改前先检查相关文件、路径、脚本和现有输出。仅当工作区本身位于 Git 仓库中时，才检查相关仓库的 `git status --short`；不得据此改动或覆盖已有未提交内容。
- 只做任务要求的最小改动。禁止破坏性操作，也不得修改、移动或删除工作区外的内容。
- 写入前，必须解析目标路径及其已存在父目录的规范路径，确认结果均位于 `/Users/youzhi/workspace/blender space/` 内；默认拒绝路径中任何符号链接。

## 目录职责

以下目录是按需约定，不代表已经存在：

- `scripts/`：经检查、可重复执行的 `bpy` Python 脚本。
- `references/`：参考图、尺寸图和材质说明等输入资料。
- `models/`：Blender 工程文件（`.blend`）。
- `renders/`：用于视觉检查或交付的渲染图。
- `exports/`：`.glb`、`.fbx`、`.obj`、`.stl` 等交换文件。

## 资产与覆盖规则

- 仅对由脚本生成的资产，把对应 `bpy` 脚本视为可复现的源文件；人工编辑的 `.blend`、用户素材和手工调整结果必须保留，不能假定可由脚本重建。
- 默认生成新文件，并使用明确名称或版本后缀，例如 `_v002`。覆盖任何已有 `.blend`、渲染图或导出文件前，必须先获得用户明确批准。
- 运行前检查目标路径，避免陈旧产物被误报为本次结果；不得以删除用户资产的方式“清理”输出。

## 脚本与依赖安全

- 仅运行已阅读并确认用途、输入和输出路径的工作区脚本。来自不可信 `.blend` 的未知嵌入 Python 或自动运行脚本不得启用或执行；检查此类文件时应保持自动执行关闭。
- 不得全局安装 Python 包、Blender 插件或系统依赖。确需依赖时，先说明用途和影响，并等待用户批准工作区内的隔离方案。

## 执行与验证

- 后台脚本使用以下命令形式，所有含空格的路径必须加引号：

```sh
"/Applications/Blender.app/Contents/MacOS/Blender" --background --python "/Users/youzhi/workspace/blender space/scripts/create_model.py"
```

- 保留完整标准输出和标准错误，检查 Python traceback，并记录 Blender 进程退出码。退出码非零即失败；退出码为零也不能单独证明成功。
- 成功必须同时满足：本次预期输出路径正确、文件存在且大小大于零。不得用旧文件、日志中的成功字样或部分生成结果代替检查。
- 涉及外观、材质、构图或几何形态的任务，必须实际打开并检查新生成的预览图；未进行视觉检查时不得宣称视觉结果合格。

## 交付报告

最终报告保持简洁，列出修改或生成的文件、执行命令与退出码、非空输出检查、视觉检查结果，以及未验证事项或风险。
